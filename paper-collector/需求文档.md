# 试卷截图错题整理工具 — 需求文档

## 1. 项目概述

### 1.1 项目名称
**Wrong Answers Collector**（试卷截图错题整理工具）

### 1.2 项目目标
对试卷截图进行 OCR 识别，自动提取题目内容并整理成 Word 文档。支持两种核心模式：
- **笔迹擦除模式**：清除试卷图片上的手写笔迹，输出清洁图片
- **OCR 识别模式**：识别图片中的题目，生成 Word 文档（支持"全部题目"与"仅错题"两种子模式）

---

## 2. 功能需求

### 2.1 输入
| 项目 | 说明 |
|------|------|
| 输入形式 | 命令行参数 |
| 输入内容 | 包含试卷截图的文件夹路径 |
| 支持格式 | `.png`, `.jpg`, `.jpeg`, `.bmp`, `.tiff`, `.webp` |

### 2.2 运行模式

#### 2.2.1 模式一：笔迹擦除（`--erase`）
- **功能**：对手机拍摄或扫描的试卷图片执行笔迹擦除与文档清洁化，输出接近“干净扫描件”的图片
- **输出位置**：输入文件夹下的 `cleaned/` 子目录
- **输出命名**：原文件名 + `_cleaned` 后缀
- **输入图片特征**：原图可能存在拍摄阴影、纸张偏灰/偏绿、透页痕迹、红笔批改、黑色手写答案、草稿演算、勾画圈注、手机拍摄黑边和轻微倾斜
- **目标效果**：参考 `笔迹擦除效果图.png`，背景趋近纯白，印刷题干、题号、图形和页码清晰保留，学生手写答案与批改痕迹尽量移除

##### 2.2.1.1 必须擦除的内容
- **红色批改笔迹**：红色对勾、叉号、圈画、分数、批注文字、红色长线
- **黑色/深色手写答案**：填空横线上的数字、文字、单位、选择题答案、公式草稿
- **题目旁手写标记**：左侧“错”等手写标记、题干周围的圈画、划线、下划线外的答案笔迹
- **底部草稿区**：第 21 题下方的大块演算、草稿公式、手写说明、红色圈画和长划线
- **拍摄噪声**：页面边缘黑边、局部阴影、灰雾、透页残影和零散污点

##### 2.2.1.2 必须保留的内容
- **印刷题干正文**：所有题目文本、数字、单位、分数、数学符号必须尽量完整保留，不应出现断笔、缺字、模糊或过度变淡
- **题号与题号圆标**：左侧 7~21 等题号、灰色/橙色圆形题号标识必须保留，不能被当作手写或边缘噪声擦除
- **填空横线**：题目中的横线、括号、空格线应保留，只擦除横线附近的手写答案
- **印刷图形**：第 11、18、20 题配图中的圆、矩形、阴影区域、辅助线、图号说明必须保留
- **章节标题和页码**：“二、填空题”“三、计算题”、底部页码等印刷内容应保留

##### 2.2.1.3 默认处理管线（`--erase-backend doc-image-tool`）
1. **彩色笔迹检测与擦除**：在 HSV 颜色空间检测高饱和红色笔迹，对红色批改线、圈注、分数和批注执行白填；避免误伤试卷中偏橙色的印刷题号和图形底色
2. **背景归一化**：消除手机拍摄造成的灰底、纸张发黄/发绿、阴影和透页痕迹，使纸面趋近纯白
3. **文档漂白**：使用 Sauvola 自适应阈值增强文字和图形边界，提升印刷内容对比度
4. **笔记去噪美化**：集成 Doc-Image-Tool 的 `HandwritingDenoisingBeautifying`，通过 K-Means 颜色量化减少背景噪声和浅色残影
5. **填空线邻域擦除**：识别题目中的横向填空线，限定在横线附近的窄带内擦除小型手写连通域；保留横线本身，避免擦掉题干文字
6. **行尾填空擦除**：对“左侧为印刷题干、右侧为填空答案”的行执行局部白填，只处理答案区域，不整行擦除
7. **底部草稿区擦除**：识别第 21 题下方的大面积手写演算区域并白填；保留第 21 题题干本身
8. **题号栏保护**：对左侧题号栏执行保护/恢复，避免题号圆标被漂白或边缘清理误删
9. **保守文字增强**：只做灰度拉伸和背景白化，不使用会删除细笔画的强形态学开运算，优先保证题干完整性
10. **边缘清理**：清理最外侧拍摄黑边和明显污迹，但不能覆盖题号栏

##### 2.2.1.4 外部模型后端
- 支持通过 `--erase-backend deeplabv3plus` 接入 `HandWritingEraser-Pytorch`，传入 `--model-repo` 与 `--model-checkpoint` 后执行模型推理
- 支持通过 `--erase-backend dis` 接入 DIS 类手写分割模型，适合输出手写区域 mask 后白填
- 支持通过 `--erase-backend erasenet` 接入 EraseNet/Paddle 类生成式擦除模型，适合直接输出清洁图
- 支持通过 `--erase-backend torchscript` 直接加载 TorchScript 或完整 PyTorch 模型，模型输出约定为手写区域 mask
- 外部模型失败时应回退到默认 `doc-image-tool` 管线，不应直接输出未处理或更差的结果

##### 2.2.1.5 质量验收标准
- **题干完整性优先**：不得为了擦除答案而明显损坏印刷题干、题号或配图；若擦除与保留冲突，优先保留印刷内容
- **题号保留**：左侧题号 7~21 应可辨认，圆形题号底纹可转为灰度，但不能整体消失
- **答案擦除**：填空横线上的手写答案应大部分消失，残留不得影响 OCR 识别题干
- **图形保留**：第 11、18、20 题图形轮廓、阴影区域、图号文字应可辨认
- **底部清洁**：第 21 题题干保留，题干下方大块草稿和红色批改应尽量清空
- **背景观感**：输出图片应接近白底黑字扫描件，减少灰雾、阴影和透页，但不得因过度二值化导致文字断裂

#### 2.2.2 模式二：OCR 识别
- **功能**：对每张图片执行 OCR 识别，提取题目并生成 Word 文档
- **子模式**：

| `--mode` | 说明 |
|----------|------|
| `all` | 提取所有题目（默认） |
| `wrong` | 仅提取被标记为错误的题目 |

- **输出位置**：输入文件夹下，文件名为 `{output_name}.docx`

### 2.3 OCR 文本识别

#### 2.3.1 OCR 引擎
- 使用 **Pix2Text (P2T)** 作为 OCR 引擎
- 采用**单例模式**管理 Pix2Text 实例，避免重复加载模型
- 配置：`box_thresh=0.3`, `unclip_ratio=2.0`

#### 2.3.2 文本清洗
OCR 识别结果经以下清洗流程：

| 步骤 | 处理内容 |
|------|----------|
| 1 | 移除替换字符（`U+FFFD`） |
| 2 | 移除零宽字符、双向控制字符（`U+200B`~`U+206F`） |
| 3 | 移除控制字符（`U+0000`~`U+001F`、`U+007F`~`U+009F`） |
| 4 | 移除私有区字符（PUA） |
| 5 | Unicode 规范化（NFC） |
| 6 | 擦除手写答题笔迹（填空答案、选择项等） |
| 7 | 清理 OCR 噪声（重复标点、孤立字母、多余空格） |
| 8 | 移除长无意义字母串（≥15 个连续字母） |
| 9 | 过滤垃圾行（伪中文乱码、无意义符号、OCR 幻想文本） |

#### 2.3.3 文本级手写笔迹擦除规则
通过正则表达式识别并擦除 OCR 文本中的手写内容：

| 规则 | 说明 |
|------|------|
| 填空答案 | 括号/方框内的短内容（≤5 字符，不含中文）→ 清空保留括号 |
| 答案提示后 | "答：""答案："等提示词后的短内容 → 替换为占位符 |
| 孤立选择项 | 单独成行的 A/B/C/D 字母 → 移除 |
| 手写标记 | 星号标记、连续下划线、对勾(√)、斜线 → 替换为空格 |

### 2.4 错题识别

#### 2.4.1 题号格式识别
支持以下题号格式：

| 格式 | 示例 |
|------|------|
| `数字 + 标点` | `1.` `1、` `1)` `1]` |
| `第N题` | `第1题` `第 1 题` |
| `括号数字` | `(1)` `1` |
| `方括号数字` | `【1】` |
| `圆圈数字` | `① ② ③ ④ ⑤ ⑥ ⑦ ⑧ ⑨ ⑩` |
| `字母 + 标点` | `A.` `B、` `C)` |

#### 2.4.2 错题标记识别
支持识别以下标记符号：

| 类别 | 标记 |
|------|------|
| 中文标记 | 错、错误、答错、不对、不正确 |
| 符号标记 | ×、✗、✘、❌ |
| 英文标记 | wrong、false、error |
| 描述性标记 | 三角、红色、红叉、叉号、打叉、划掉、删除、取消 |

#### 2.4.3 错题定位策略
三种匹配模式：

| 模式 | 说明 |
|------|------|
| 题号 + 下行标记 | 题号行后紧接错题标记行 |
| 标记 + 下行题号 | 错题标记行后紧接题号行 |
| 同行标记 | 题号与标记在同一行 |

去重：按题号去重，避免同一题被重复收录。

### 2.5 Word 文档生成

- 格式：`.docx`
- 默认字体：宋体，11pt
- 结构：
  - 文档标题：居中"题目整理"
  - 概要行：`共整理 N 道题目`
  - 每题以"题目 N"为一级标题，下方为题文段落
- 所有文本经文本清洗后再写入文档

## 3. 技术架构

### 3.1 依赖

| 库 | 用途 |
|----|------|
| `pix2text` | OCR 文字、公式、表格识别 |
| `python-docx` | Word 文档生成 |
| `Pillow` | 图片处理 |
| `opencv-python (cv2)` | 图片处理、笔迹擦除、连通域分析 |
| `numpy` | 数值计算 |
| `unicodedata` | Unicode 规范化 |

### 3.2 集成外部模块
- **Doc-Image-Tool**：默认笔迹擦除管线，提供文档漂白（`DocBleach.sauvola_threshold`）和笔记去噪美化（`HandwritingDenoisingBeautifying.docscan_main`）
- **HandWritingEraser-Pytorch**：DeepLabV3+ 手写分割后端，可通过权重文件执行模型推理
- **Handwriting-Removal-DIS**：DIS 手写区域分割后端，可输出手写 mask 并做白填
- **bdpan_erase_competition**：EraseNet/Paddle 生成式擦除后端，可输出清洁图片

### 3.3 关键设计决策

| 决策 | 原因 |
|------|------|
| Pix2Text 单例模式 | 避免重复加载模型，提升性能与识别稳定性 |
| 强制 UTF-8 输出 | 避免控制台乱码 |
| 禁用 Paddle 网络检查 | 使用本地模型缓存，提升加载速度 |
| 保守擦除策略 | 题干完整性优先，避免为了擦答案而破坏题干、题号和配图 |
| 分区域笔迹擦除 | 题干区、填空区、题号栏、配图区、底部草稿区使用不同擦除力度 |
| 题号栏保护 | 题号圆标容易被漂白/边缘清理误删，需要单独保护或恢复 |
| 模型失败回退 | 外部权重或依赖缺失时回退到 `doc-image-tool` 管线，保证有可用输出 |

## 4. 命令行接口

### 4.1 基本用法

```bash
# 笔迹擦除模式
python exam_scanner.py <图片文件夹> --erase

# OCR 识别所有题目
python exam_scanner.py <图片文件夹> <输出文件名> --mode all

# OCR 识别仅错题
python exam_scanner.py <图片文件夹> <输出文件名> --mode wrong
```

### 4.2 参数说明

| 参数 | 必需 | 说明 |
|------|------|------|
| `folder_path` | 是 | 包含试卷截图的文件夹路径 |
| `output_name` | OCR 模式必需 | Word 文档名称（不含扩展名） |
| `--erase` | 否 | 启用笔迹擦除模式 |
| `--erase-backend` | 否 | 笔迹擦除后端，默认 `doc-image-tool`，可选 `opencv` 或外部模型后端 |
| `--model-repo` | 外部模型后端可选 | 第三方模型仓库路径 |
| `--model-checkpoint` | 外部模型后端可选 | 模型权重路径，如 `.pth`、`.pd`、`.pt` |
| `--model-command` | 外部模型后端可选 | 第三方预测命令模板，可用 `{input}`、`{output}`、`{repo}`、`{checkpoint}`、`{device}` 等占位符 |
| `--model-device` | 否 | 模型推理设备，默认 `auto` |
| `--mask-threshold` | 否 | mask 模型输出二值化阈值，默认 `0.5` |
| `--mode` | 否 | 整理模式：`all`（全部题目）或 `wrong`（仅错题），默认 `all` |

## 5. 非功能需求

| 需求 | 说明 |
|------|------|
| 编码兼容 | 全程 UTF-8，避免中文乱码 |
| 乱码检测 | 对 OCR 结果进行乱码检测并给出警告 |
| 异常处理 | 各阶段出错不中断整体流程，跳过失败项继续处理 |
| 进度反馈 | 实时输出处理进度（`[n/N]`）和阶段信息 |
| 容错性 | 漂白/去噪失败时自动跳过该阶段，使用上一阶段结果 |
