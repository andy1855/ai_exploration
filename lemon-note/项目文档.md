### 关键路径说明

- **`base: '/lemon-note/'`** — Vite 配置了 base 路径，生产构建所有资源路径以 `/lemon-note/` 开头，适配 Nginx 子路径部署
- 开发时用 `npm run dev` 在 `localhost:3000` 运行，base 路径不影响开发模式

## 3. 核心架构

## 4. 组件说明

### 4.1 `App.tsx`

- **职责**: 根组件，初始化主题，渲染 Sidebar + Editor 布局
- **关键逻辑**: 监听 `preferences.theme` 变化，同步设置 `document.documentElement` 的 `data-theme` 属性

### 4.2 `Sidebar.tsx`

| 功能 | 说明 |
|------|------|
| 主题切换 | 点击顶部的 🌙/☀ 按钮切换亮/暗色模式 |
| 搜索文稿 | 实时过滤，按标题和内容匹配 |
| 新建文稿 | 创建后自动选中并进入编辑状态 |
| 新建分组 | 创建根级分组 |
| 分组管理 | 嵌套分组、折叠/展开、双击重命名、删除 |
| 文稿管理 | 点击选中、悬停显示删除按钮 |
| 空状态 | 无数据时显示引导提示 |

**`GroupItem` 子组件** — 递归渲染嵌套分组和其下的文稿列表。每层缩进递增 `16px`。

### 4.3 `Editor.tsx`

| 功能 | 说明 |
|------|------|
| 空状态引导 | 未选中文稿时显示引导页，含新建按钮 |
| 标题编辑 | 大号输入框，自动保存（500ms 防抖） |
| Markdown 编辑 | 基于 `@uiw/react-md-editor`，支持编辑/实时预览切换 |
| 字数统计 | 自动统计中英文混排字数 |
| 预览切换 | 工具栏 👁 按钮切换 `live`(编辑+预览) / `edit`(仅编辑) |
| 专注模式 | 工具栏 ⛶ 按钮，隐藏工具栏，扩大编辑区域 |
| 删除文稿 | 工具栏 🗑 按钮，确认后删除 |
| 自动保存 | 任何修改 500ms 防抖后自动写入 store + localStorage |

**MDEditor 模式说明**:

| `preview` 值 | 效果 |
|:---|:---|
| `'live'` | 同时显示编辑区和预览区（默认） |
| `'edit'` | 仅显示编辑区 |
| `'preview'` | 仅显示预览区（只读） |

## 5. 状态管理

使用 **Zustand** 管理全局状态，定义在 `useNoteStore.ts`。

### 5.1 State 结构

```typescript
interface NoteState {
  // 数据
  sheets: Sheet[];           // 所有文稿
  groups: Group[];           // 所有分组
  selectedSheetId: string | null;
  selectedGroupId: string | null;
  preferences: AppPreferences;
  searchQuery: string;

  // Actions — Sheet
  createSheet(groupId?: string): string;
  updateSheet(id: string, updates: Partial<Sheet>): void;
  deleteSheet(id: string): void;
  moveSheet(id: string, groupId: string | null): void;
  selectSheet(id: string | null): void;

  // Actions — Group
  createGroup(name?: string, parentId?: string | null): string;
  updateGroup(id: string, updates: Partial<Group>): void;
  deleteGroup(id: string): void;
  selectGroup(id: string | null): void;

  // Actions — Preferences
  updatePreferences(updates: Partial<AppPreferences>): void;

  // Actions — Search
  setSearchQuery(query: string): void;

  // Derived data
  getFilteredSheets(): Sheet[];
  getSheetsByGroup(groupId: string | null): Sheet[];
  getChildGroups(parentId: string | null): Group[];
}
```

### 5.2 数据模型

```typescript
interface Sheet {
  id: string;
  title: string;
  content: string;
  createdAt: number;       // 时间戳
  updatedAt: number;       // 时间戳
  groupId: string | null;  // 所属分组 (null = 未分组)
  pinned?: boolean;
  wordCount?: number;
}

interface Group {
  id: string;
  name: string;
  icon?: string;
  color?: string;
  parentId: string | null;  // 父级分组 (null = 根级)
  order: number;            // 排序
  collapsed?: boolean;
}

interface AppPreferences {
  theme: 'light' | 'dark' | 'system';
  sidebarWidth: number;       // 280
  editorFontSize: number;     // 16
  showPreview: boolean;       // 默认 true (live 模式)
  focusMode: boolean;         // 默认 false
  showWordCount: boolean;     // 默认 true
}
```

## 6. 数据持久化

| 存储键 | 内容 | 存储方式 |
|--------|------|----------|
| `lemon-note-data` | `{ sheets, groups }` | `JSON.stringify` → localStorage |
| `lemon-note-preferences` | `AppPreferences` | `JSON.stringify` → localStorage |

- 每次状态变更自动通过 `saveData()` / `savePreferences()` 写入 localStorage
- 应用启动时通过 `loadData()` / `loadPreferences()` 读取
- 目前**纯前端存储**，无后端/云端同步

## 7. 样式系统

### 7.1 主题变量

使用 CSS 自定义属性实现亮/暗双主题：

- `[data-theme='light']` — 浅色模式（默认）
- `[data-theme='dark']` — 深色模式

关键变量分类：

| 类别 | 变量前缀 | 示例 |
|------|----------|------|
| 背景色 | `--bg-*` | `--bg-primary`, `--bg-secondary` |
| 文字色 | `--text-*` | `--text-primary`, `--text-secondary` |
| 边框 | `--border-*` | `--border-color` |
| 强调色 | `--accent*` | `--accent: #2563eb` |
| 阴影 | `--shadow-*` | `--shadow-sm` |
| 圆角 | `--radius-*` | `--radius-md: 8px` |
| 字体 | `--font-*` | `--font-sans`, `--font-mono` |

### 7.2 组件尺寸变量

```css
--sidebar-width: 280px;
--header-height: 48px;
--toolbar-height: 44px;
--editor-font-size: 16px;
```

### 7.3 Markdown 编辑器样式

MDEditor 的 `.w-md-editor-*` 选择器被深度定制，覆盖了：
- 编辑器背景透明以适配主题
- 语法高亮 token 颜色
- 预览区 (`.wmde-markdown`) 样式：标题、引用、代码块、表格等
- 隐藏默认工具栏

## 8. 部署流程

### 8.1 本地开发

```bash
cd lemon-note
npm install
npm run dev      # → http://localhost:3000
```

### 8.2 服务器部署

```bash
# 1. 提交代码到 GitHub
git add lemon-note/
git commit -m "..."
git push

# 2. SSH 到服务器，拉取 + 构建
ssh aliyun
cd /root/ai_exploration
git pull
cd lemon-note
npm install
npm run build

# 3. 复制构建产物到 Web 目录
cp -r dist/* /var/www/lemon-note/
```

### 8.3 Nginx 配置

部署在子路径 `/lemon-note/` 下，参考 `todo-list.conf`：

```nginx
location /lemon-note/ {
    alias /var/www/lemon-note/;
    index index.html;
    try_files $uri $uri/ /lemon-note/index.html;
}
```

> **注意**: Vite 已配置 `base: '/lemon-note/'`，构建产物的所有资源路径会自动添加该前缀。

## 9. 已知问题

1. **构建警告**: JS 产物约 1.2MB（含 MDEditor 库），建议后续用动态导入拆分
2. **无后端存储**: 数据仅存 localStorage，清除浏览器数据会丢失
3. **无协同编辑**: 目前仅单用户使用
4. **`index.html` 中 favicon 路径**: 硬编码为 `/favicon.svg`，子路径部署下应为 `/lemon-note/favicon.svg`（构建时 Vite 会自动处理）
