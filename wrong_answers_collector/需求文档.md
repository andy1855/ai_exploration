# 试卷截图错题整理工具 — 需求文档

## 1. 项目概述

### 1.1 项目名称
**Wrong Answers Collector**（试卷截图错题整理工具）

### 1.2 项目目标
对试卷截图进行 OCR 识别，自动提取题目内容并整理成 Word 文档。支持两种核心模式：
- **笔迹擦除模式**：清除试卷图片上的手写笔迹，输出清洁图片
- **OCR 识别模式**：识别图片中的题目，生成 Word 文档（支持"全部题目"与"仅错题"两种子模式）

---

## 2. 功能需求

### 2.1 输入
| 项目 | 说明 |
|------|------|
| 输入形式 | 命令行参数 |
| 输入内容 | 包含试卷截图的文件夹路径 |
| 支持格式 | `.png`, `.jpg`, `.jpeg`, `.bmp`, `.tiff`, `.webp` |

### 2.2 运行模式

#### 2.2.1 模式一：笔迹擦除（`--erase`）
- **功能**：对每张图片执行手写笔迹擦除，输出清洁图片
- **输出位置**：输入文件夹下的 `cleaned/` 子目录
- **输出命名**：原文件名 + `_cleaned` 后缀
- **处理管线**（三阶段）：
  1. **文档漂白**：Sauvola 自适应阈值二值化，白底黑字
  2. **笔记去噪美化**：K-Means 颜色量化（集成 Doc-Image-Tool 的 `HandwritingDenoisingBeautifying` 模块）
  3. **精细擦除**：连通域分析 + 图像修复（`cv2.inpaint`），按"题干区"与"答题区"分区域处理

#### 2.2.2 模式二：OCR 识别
- **功能**：对每张图片执行 OCR 识别，提取题目并生成 Word 文档
- **子模式**：

| `--mode` | 说明 |
|----------|------|
| `all` | 提取所有题目（默认） |
| `wrong` | 仅提取被标记为错误的题目 |

- **输出位置**：输入文件夹下，文件名为 `{output_name}.docx`

### 2.3 OCR 文本识别

#### 2.3.1 OCR 引擎
- 使用 **Pix2Text (P2T)** 作为 OCR 引擎
- 采用**单例模式**管理 Pix2Text 实例，避免重复加载模型
- 配置：`box_thresh=0.3`, `unclip_ratio=2.0`

#### 2.3.2 文本清洗
OCR 识别结果经以下清洗流程：

| 步骤 | 处理内容 |
|------|----------|
| 1 | 移除替换字符（`U+FFFD`） |
| 2 | 移除零宽字符、双向控制字符（`U+200B`~`U+206F`） |
| 3 | 移除控制字符（`U+0000`~`U+001F`、`U+007F`~`U+009F`） |
| 4 | 移除私有区字符（PUA） |
| 5 | Unicode 规范化（NFC） |
| 6 | 擦除手写答题笔迹（填空答案、选择项等） |
| 7 | 清理 OCR 噪声（重复标点、孤立字母、多余空格） |
| 8 | 移除长无意义字母串（≥15 个连续字母） |
| 9 | 过滤垃圾行（伪中文乱码、无意义符号、OCR 幻想文本） |

#### 2.3.3 文本级手写笔迹擦除规则
通过正则表达式识别并擦除 OCR 文本中的手写内容：

| 规则 | 说明 |
|------|------|
| 填空答案 | 括号/方框内的短内容（≤5 字符，不含中文）→ 清空保留括号 |
| 答案提示后 | "答：""答案："等提示词后的短内容 → 替换为占位符 |
| 孤立选择项 | 单独成行的 A/B/C/D 字母 → 移除 |
| 手写标记 | 星号标记、连续下划线、对勾(√)、斜线 → 替换为空格 |

### 2.4 错题识别

#### 2.4.1 题号格式识别
支持以下题号格式：

| 格式 | 示例 |
|------|------|
| `数字 + 标点` | `1.` `1、` `1)` `1]` |
| `第N题` | `第1题` `第 1 题` |
| `括号数字` | `(1)` `1` |
| `方括号数字` | `【1】` |
| `圆圈数字` | `① ② ③ ④ ⑤ ⑥ ⑦ ⑧ ⑨ ⑩` |
| `字母 + 标点` | `A.` `B、` `C)` |

#### 2.4.2 错题标记识别
支持识别以下标记符号：

| 类别 | 标记 |
|------|------|
| 中文标记 | 错、错误、答错、不对、不正确 |
| 符号标记 | ×、✗、✘、❌ |
| 英文标记 | wrong、false、error |
| 描述性标记 | 三角、红色、红叉、叉号、打叉、划掉、删除、取消 |

#### 2.4.3 错题定位策略
三种匹配模式：

| 模式 | 说明 |
|------|------|
| 题号 + 下行标记 | 题号行后紧接错题标记行 |
| 标记 + 下行题号 | 错题标记行后紧接题号行 |
| 同行标记 | 题号与标记在同一行 |

去重：按题号去重，避免同一题被重复收录。

### 2.5 Word 文档生成

- 格式：`.docx`
- 默认字体：宋体，11pt
- 结构：
  - 文档标题：居中"题目整理"
  - 概要行：`共整理 N 道题目`
  - 每题以"题目 N"为一级标题，下方为题文段落
- 所有文本经文本清洗后再写入文档

## 3. 技术架构

### 3.1 依赖

| 库 | 用途 |
|----|------|
| `pix2text` | OCR 文字、公式、表格识别 |
| `python-docx` | Word 文档生成 |
| `Pillow` | 图片处理 |
| `opencv-python (cv2)` | 图片处理、笔迹擦除、连通域分析 |
| `numpy` | 数值计算 |
| `unicodedata` | Unicode 规范化 |

### 3.2 集成外部模块
- **Doc-Image-Tool**：笔迹擦除的文档漂白（`DocBleach.sauvola_threshold`）和笔记去噪美化（`HandwritingDenoisingBeautifying.docscan_main`）

### 3.3 关键设计决策

| 决策 | 原因 |
|------|------|
| Pix2Text 单例模式 | 避免重复加载模型，提升性能与识别稳定性 |
| 强制 UTF-8 输出 | 避免控制台乱码 |
| 禁用 Paddle 网络检查 | 使用本地模型缓存，提升加载速度 |
| 分区域笔迹擦除 | 题干区与答题区擦除力度不同，保护印刷文字 |

## 4. 命令行接口

### 4.1 基本用法

```bash
# 笔迹擦除模式
python exam_scanner.py <图片文件夹> --erase

# OCR 识别所有题目
python exam_scanner.py <图片文件夹> <输出文件名> --mode all

# OCR 识别仅错题
python exam_scanner.py <图片文件夹> <输出文件名> --mode wrong
```

### 4.2 参数说明

| 参数 | 必需 | 说明 |
|------|------|------|
| `folder_path` | 是 | 包含试卷截图的文件夹路径 |
| `output_name` | OCR 模式必需 | Word 文档名称（不含扩展名） |
| `--erase` | 否 | 启用笔迹擦除模式 |
| `--mode` | 否 | 整理模式：`all`（全部题目）或 `wrong`（仅错题），默认 `all` |

---

## 5. 非功能需求

| 需求 | 说明 |
|------|------|
| 编码兼容 | 全程 UTF-8，避免中文乱码 |
| 乱码检测 | 对 OCR 结果进行乱码检测并给出警告 |
| 异常处理 | 各阶段出错不中断整体流程，跳过失败项继续处理 |
| 进度反馈 | 实时输出处理进度（`[n/N]`）和阶段信息 |
| 容错性 | 漂白/去噪失败时自动跳过该阶段，使用上一阶段结果 |
